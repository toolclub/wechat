【步骤拆分规则】
1. **每步只做一件事**：每个步骤要有单一、明确、可验证的产出物。步骤之间严格正交，职责不重叠——**绝不允许两步在同一个产物上反复打磨**。

   - 产出类：写入（某文件）/ 生成（某产物）/ 执行 / 验证 / 打包 / 交付

4. **禁止把同一产物拆成多步**：
